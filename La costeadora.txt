Descripción General del Proyecto
"Studio Financiero" es una Aplicación Web Progresiva (PWA) interactiva diseñada para la realidad del emprendedor creativo moderno (artesanos, muralistas, diseñadores, freelancers). El objetivo de la aplicación es traducir conceptos financieros complejos (punto de equilibrio, flujo de caja, costeo de proyectos y productos) a un lenguaje visual, amigable y altamente intuitivo.

El sistema debe operar bajo la premisa de que el creativo no está en un escritorio. La app está pensada para usarse con una sola mano, en medio de un taller o en la calle, permitiendo registrar datos financieros sin fricción y ofreciendo un panorama claro de la salud del negocio de un solo vistazo.

2. Pila Tecnológica
La aplicación mantiene un ecosistema robusto, pero optimizado para rendimiento móvil:

Framework Core: React 18 con TypeScript y Vite (Configurado como PWA para instalación en celulares).

Enrutamiento: React Router v7.

Estilos y UI: TailwindCSS v4, enfocado en utilidades responsivas. Se integran componentes accesibles de Radix UI (ideales para modales y menús táctiles) y animaciones fluidas de Framer Motion.

Gráficos: Recharts para la visualización de datos (optimizados para pantallas estrechas, priorizando interactividad táctil).

Gestión de Estado: Context API (FinancialDataContext) o Zustand, diseñado para manejar estados globales complejos que ahora incluyen no solo productos, sino también "Proyectos a medida", "Inventario global" y "Flujos de caja fraccionados".

3. Filosofía del Sistema y Reglas de Diseño (UX/UI)
Para que Antigravy desarrolle correctamente el front-end, debe adherirse estrictamente a los siguientes principios de diseño:

Mobile-First Absoluto: Todo el diseño parte de una pantalla de celular (aprox. 375px a 430px de ancho). Los elementos deben ser fáciles de tocar (tap targets grandes), y la navegación principal debe estar al alcance del pulgar (ej. en la parte inferior). La versión de escritorio será una expansión fluida de esta vista móvil, no al revés.

Divulgación Progresiva (Progressive Disclosure): Es la regla de oro para la interfaz. Las pantallas principales deben ser extremadamente limpias, mostrando solo los "números grandes", gráficos resumidos e íconos expresivos (Lucide-react). La complejidad (tablas densas, formularios largos, desgloses matemáticos) debe estar oculta por defecto y revelarse solo cuando el usuario toca un botón de "Ver detalles", "Analizar" o abre un panel expansible.

Diseño "Cristalino" y Reductor de Ansiedad: Mantener el uso de fondos oscuros elegantes (Dark Mode preferido) con efectos de cristalismo (backdrop-blur). Los colores deben usarse estratégicamente: semáforos visuales (verde, rojo, amarillo) para indicar salud financiera, evitando paredes de texto que generen "ansiedad matemática".

Flexibilidad del Negocio Creativo (Proyectos vs. Productos): La lógica interna debe soportar dos realidades:

El Catálogo: Productos fijos estandarizados (ej. una mochila, una lámina).

El Proyecto a Medida: Servicios variables (ej. pintar un mural), que requieren costear tiempo (horas/días), materiales específicos del inventario, viáticos, e hitos de cobro (anticipos).

Antigravity debe configurar Tailwind CSS v4 y el CSS global con las siguientes especificaciones técnicas exactas:

4.1. Sistema Tipográfico (CSS Variables)
El HTML de referencia define tres familias tipográficas críticas que deben mapearse en Tailwind. Se deben definir en el archivo CSS principal (ej. globals.css) como variables nativas:

--font-family-disp (Display): Para Headers gigantes, números de impacto y KPIs. Debe ser una fuente grotesca sans-serif muy pesada y con tracking ajustado (ej. Clash Display, Space Grotesk o similar a Haas Grot).

--font-family-text (Body): Para la lectura general, etiquetas y tooltips. Sans-serif limpia y legible (ej. Inter o Geist).

--font-family-mono (Monospace): OBLIGATORIO para todos los datos financieros, tablas, matrices y el "Diario". (ej. Martian Mono, JetBrains Mono).

4.3. Arquitectura de Componentes de UI
Antigravity debe crear componentes base (usando Radix UI para la accesibilidad) con las siguientes reglas visuales:

Botones (Pill-shape): Formas completamente redondeadas (rounded-full). Sin drop-shadows suaves. Si llevan sombra, debe ser sólida/dura (box-shadow: 2px 2px 0px #111118) o carecer de ella.

Paneles y Tarjetas (Componente Panel): Las tarjetas de proyectos o insights deben tener bordes redondeados (rounded-xl o rounded-2xl).

4.5. Sistema de Animaciones (Framer Motion)
Las animaciones no deben ser lineales ni "fade-ins" aburridos. Deben sentirse físicas y "elásticas".

Directiva Framer Motion: Para la entrada progresiva (Progressive Disclosure) de elementos de detalle (ej. abrir el desglose de Gastos Fijos), usar la configuración type: "spring".

Ejemplo de variante estándar: initial={{ opacity: 0, y: 30, rotate: -2 }} a animate={{ opacity: 1, y: 0, rotate: 0 }}.

Transición recomendada: { stiffness: 300, damping: 20, mass: 1 }.

4. Núcleo Matemático (Lógica de Estado Global)
Implementa las siguientes fórmulas exactas en el Estado Global o hooks (ej. useProjectMath):

Fórmulas de Costeo:

productionTotal = SUM(prod.quantity * prod.unitPrice)

distributionTotal = SUM(dist.quantity * dist.unitPrice)

investmentPerUnit = estimatedUnits > 0 ? initialInvestment / estimatedUnits : 0

productCost = productionTotal + distributionTotal + investmentPerUnit

Margen de Ganancia (El KPI más importante):

profitMargin = ((sellingPrice - productCost) / sellingPrice) * 100

Punto de Equilibrio (Distribución Equitativa):

Importante: El sistema NO usa margen de contribución ponderado. Los gastos fijos se dividen por igual entre proyectos activos.

productFixedShare = totalFixedExpenses / products.length

unitContribution = sellingPrice - productCost

unitsNeeded = Math.ceil(productFixedShare / unitContribution)

revenueNeeded = unitsNeeded * sellingPrice

Valor de Autoría (UI Marker): No afecta matemáticamente al productCost. Es un booleano (affectedByAuthorship: true) que aplica un borde neón y un ícono (⭐) a ese ítem de costo en la UI para recordarle al creativo que cobre su talento.
